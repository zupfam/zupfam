# Dishgram Constitution

## Core Principles

### I. Mobile-First Design
The application MUST be designed for mobile devices first. The user experience and interface must be optimized for smaller screens and touch interactions, then progressively enhanced for tablets and desktops. This ensures a quality experience for the majority of users.

### II. Instagram-Like Experience
The user interface MUST be highly visual, intuitive, and centered around media. The core user journey should be simple, engaging, and encourage browsing and discovery, mirroring the user experience of platforms like Instagram.

### III. Static Web Application
The application MUST be delivered as a static web app. All assets (HTML, CSS, JavaScript, images) are to be pre-built and served from a Content Delivery Network (CDN). This approach prioritizes performance, security, and scalability, while minimizing server-side dependencies.

### IV. Vendor Branding Platform
The primary purpose of this application is to serve as a branding and discovery platform for local food vendors. All features and design decisions MUST support this goal, enabling vendors to showcase their brand and products effectively.

## Technology Stack

The technology stack will be determined during the planning phase, but it must be able to produce a static site.

## Development Workflow

We will follow a test-driven development (TDD) approach. All code changes must be submitted via pull requests and reviewed by at least one other team member.

## Governance

All pull requests and reviews must verify compliance with this constitution. Any deviation must be justified and documented.
